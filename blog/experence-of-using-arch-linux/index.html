Arch Linux 使用体验

26 March 2022 / 5 min read
  • linux
  • blog

为什么要用 Arch Linux？

要问为什么要用 Arch Linux，首先要问为什么要用 Linux。

如果在 Google 搜索为什么用 Linux，会得到五花八门的答案。

我为什么要用 Linux？回想第一次安装 Linux，那个时候我心底的想法应该是装逼。接触的第一个 Linux 发行版是 Ubuntu，同学推荐的。安装很方便，桌面环境还能随意定置，很炫酷。

后来我还是放弃了 Ubuntu 桌面，最主要的原因时，使用一段时间之后，总会出现一些不明所以的 error，虽然都不影响正常使用，但总会让人很不爽。正好 WSL 出现了，既然可以在使用 Windows 的同时使用 Linux，当然要用了。

那么为什么要用 Arch Linux 呢？因为使用 WSL 用的还是 Ubuntu，安装软件包还是挺麻烦，听说 Arch Linux 的自由度很高，那何不尝试一下呢。

怎么安装？

很多人说 Arch Linux 拥有强大的 wiki，确实如此，在 Arch Linux wiki 中可以找到你所需要的一切。

安装 Arch Linux，最直接的办法就是安装官方安装指南一步一步操作。官方指南的特点就是大而全，这是好事，但也会给安装带来一些干扰。当然，网络上也有很多高手写了简洁明了的安装指南，经过一番搜索，我推荐由 @ArchLinuxStudio 编写的 Arch Linux 安装使用教程。

体验如何？

Arch Linux 所有一切都可以自己选择安装，我使用的是 KDE Plasma 桌面环境，定置程度很高，完全可以按照自己习惯的方式进行改造。当然，改造或美化应适可而止。

很多人推荐这个桌面环境，我第一次尝试的时候还是有些不习惯的，比如它把终端叫做 Konsole，而不是 Console 或 Terminal，就感觉是为了 K 而 K。

真正使用之后，Arch Linux 方便易用的特点就能体现出来了。最常见的安装软件，不需要像 Ubuntu 那样需要添加 ppa，Arch Linux 只需要使用官方仓库和 AUR 就可以解决绝大多数软件的安装问题。

安装软件是首先用 sudo pacman -Ss packageName 在官方仓库找。如果找不到，就在 AUR 上找，再通过 yay -S packageName 安装。

最后的话

Arch Linux 是滚动更新的系统，很多人说容易滚挂，关于这个问题，网络上给出的建议是每天执行一遍 sudo pacman -Syyu 更新。

我在安装系统的过程中，出现了安装 [GRUB] 引导失败的问题，反复尝试了几次之后依然没有成功，最后我选择了 [rEFInd] 引导。安装完成之后因为进入不了 refind 引导界面，我又使用安装介质进入了安装环境，挂载系统盘修复 refind 引导。在这个过程中，我发现系统出现问题并不是大问题，只要使用安装介质进入安装环境挂载系统盘修复就行了。所以系统滚挂了，也不是大问题，进入安装环境修复就好了。虽然现在还没有出现滚挂的问题，但我认为这样是可以解决问题的。
