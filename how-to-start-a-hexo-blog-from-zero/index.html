如何从零开始搭建Hexo博客

2019-10-13 #notes, #Tutorial 2 min read

这篇文章介绍了如何从零开始，搭建一个基于 Hexo 的静态博客，并部署到 GitHub Pages 上，以及在 GitHub Pages 上绑定自己的域名，最终实现通过域名访问博客。

0. 准备工作

什么是 Hexo

既然要搭建 Hexo 网站，那么肯定知道什么是 Hexo。

根据官网的介绍，Hexo 是快速、简洁且高效的博客框架。快速是指它生成静态页面的速度很快，简洁是指它用 Markdown 语法构建页面，省去了自己写页面的麻烦，高效是指它部署网站效率高，一条指令即可把静态页面部署到网站上，而且还支持多种插件。

安装 nodejs 环境

搭建 Hexo 需要一个开发环境，即 nodejs，因此我们需要安装 nodejs，Windows 和 Mac 用户直接在 nodejs 官网 下载最新的 LTS 版本安装即可，Linux 用户建议通过以下两种方式之一先安装 nvm，然后通过 nvm 安装 nodejs。

安装 nvm

1# curl
2curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.0/install.sh | bash
3# or Wget:
4wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.35.0/install.sh | bash

通过 nvm 安装 nodejs

1nvm install node

安装 Git

因为后面需要把博客项目保存在 GitHub 上，以及部署博客到 Github Pages，都需要用到 Git 命令，因此必须安装 Git。

打开 https://www.git-scm.com/downloads 选择对应的操作系统，按照页面提示安装即可。

安装 Visual Studio Code 编辑器

搭建博客的时候，需要用到命令行，还需要修改博客配置文件，这些工作如果分开几个工具去做，则会比较麻烦。因此这里推荐安装 Visual Studio Code 编辑器，它集成了命令行工具，方便我们在同一个界面上完成配置修改，命令行调试等任务。

在 Visual Studio Code 官网 下载所需要的安装包。Windows、Mac、Linux 都可以下载到对应的安装包，安装即可。

1. 安装 Hexo 环境

nodejs 环境和 VS Code 编译器安装完成之后，就可以真正的开始安装 Hexo 了。

安装 Hexo

安装 Hexo 非常简单。打开命令行工具，输入如下命令：

1npm install hexo-cli -g

等待安装完成。使用命令 cd 到你想要保存博客项目的文件夹，并执行如下命令：

1hexo init blog

会执行一大堆命令，不要着急，等待命令执行完毕，这个时候你的博客项目已经创建好了，项目文件名就是上面命令里的 blog，当然你也可以起一个自己喜欢的名字，只不过文件名中间不能有空格。然后进入博客项目文件夹，执行下面的命令：

1# 进入博客项目文件夹
2cd blog
3# 安装依赖
4npm install
5# 启动服务
6hexo server

当执行完 hexo server 后，命令行窗口会显示 Hexo is running at http://localhost:4000 . Press Ctrl+C to stop 的提示，打开浏览器，输入 localhost:4000 即可看到生成的博客效果。

2. 部署网站到 GitHub Pages

要想把博客部署到 GitHub Pages 上，需要做几件事。

  1. 创建 GitHub 账号
  2. 新建 GitHub Pages 仓库
  3. 新建博客项目仓库（可选）
  4. 配置博客
  5. 部署及写作

创建 GitHub 账号

打开 GitHub 官网 github.com，注册即可。

新建 GitHub Pages 仓库

在注册 GitHub 账号时，你有一个自己的用户名，在创建 Github Pages 仓库时，需要用到这个用户名。假如你的用户名是 someone，那么你创建的 GitHub Pages 仓库名必须是 someone.github.io，如果用了其他的名字，则后续就不能通过 Hexo 进行部署。

新建博客项目仓库（可选）

操作步骤和上面一样，这个仓库是用来保存博客项目文件的，如果你不想让别人看到，也可以把这个仓库设置成私有仓库。

GitHub Pages 仓库创建完成之后，目前还不需要做什么，但博客项目仓库创建完成之后，因为需要把博客项目存储到 GitHub 上，所以必须建立两者之间的联系。

推荐的方法是创建 sshKey，具体操作请参考 向客户端添加 SSH key。添加完 ssh key 后，在本地博客项目执行 git 命令，建立本地项目与 GitHub 仓库关系。

创建项目仓库时，会有一个仓库地址链接，把地址链接改成 SSH 形式，类似这样的：git@github.com:someone/blog.git，将这个地址添加到本地的项目中，具体做法：

1# 初始化本地仓库
2git init
3# 将项目仓库地址添加到本地仓库
4git remote add origin git@github.com:someone/blog.git

这样，就建立了本地仓库与 GitHub 仓库的连接，以后就可以把本地仓库的修改推送到 GitHub 上。现在，可以把刚刚建立的博客项目先做第一次推送。执行如下命令：

1git add -A
2git commit -m "first commit"
3git push origin master

在浏览器中打开建立的项目仓库，推送的内容就会显示在上面。

配置博客

在博客项目根目录，有一个 _config.yml 的文件，这个文件就是博客的配置文件。只需要改如下配置，即可将博客推送到 GitHub Pages 上。

修改博客信息

1# Site
2title: blog #博客的名字，标题
3subtitle: #博客副标题
4description: 我的博客 #博客描述
5keywords: #关键字，
6author: yourName #作者名
7language: zh-CN #博客的默认语言
8timezone: Asia/Shanghai #时区

博客部署信息

1# Deployment
2## Docs: https://hexo.io/docs/deployment.html
3deploy:
4  type: git #部署类型
5  repo: git@github.com:someone/someone.github.io.git #GitHub Pages 仓库地址
6  branch: master #分支，必须是 master

配置完成。

部署及写作

部署

生成 Hexo 博客项目时，已经有了一篇博文，直接把它部署到 GitHub Pages 上。确保命令行定位在博客项目文件夹下，执行如下命令：

清除生成文件

1hexo clean

生成文件

1hexo g

部署文件

1hexo d

每一步操作都需要等上一步操作完成后再进行。完成后，在浏览器输入创建的 GitHub Pages 仓库名，例如 someone.github.io，就可以看到博客的页面了。

博客搭好了，以后要做的事就是，写文章，部署文章到 GitHub Pages 上。

写作

在哪里写文章呢？

在博客项目的 source 文件夹下有一个 _posts 文件夹，打开它，就能看到生成博客项目时的 Hello Word 的 markdown 文件，以后写文章写在这个文件夹里，就可以在网站上看到了。需要注意的是，博客文章是用 markdown 格式写得，文件开头 --- 围起来的部分，这是博客文章的一些关键信息，包含文章的标题、发表日期、标签等信息，详细设置可以参考 Hexo 官网文档。

写文章时，即可以在 _posts 文件夹里新建 markdown 文件，也可以用 hexo new articleName 创建文章，推荐后一种方法。因为通过 hexo new 命令创建的文章，会自动加上头部信息，而且这个头部信息也是可以配置的，配置文件在项目根目录下的 scaffolds，打开它会看到 post.md 的文件，相当于文章的模板。这个文件如何配置，可以查看 Hexo 官网文档。

3. 绑定域名

直接使用 GitHub Pages 提供的地址已经可以在互联网上访问博客了，但总感觉使用 github.io 这样的名字不太舒服。如果要绑定我们自己的域名怎么做呢？

首先，你得有一个域名。国内可以通过阿里云、腾讯云、京东云等云服务申请自己的域名，特点就是方便，不过需要备案，当然备案也很方便，很好操作。也可以在国外网站购买，流程也不是很复杂，不用备案也不能备案。

其次，有了域名，还需要把域名解析到 github.io 的服务器上。打开命令行终端，ping 一下你 GitHub 博客的地址，得到 ip。如 ping someone.github.io。记录一下这个 ip，把域名解析到这个 ip 上。国内云服务提供解析，按照正常解析流程解析即可。国外申请的域名，域名网站也会提供解析，也可以使用 cloudflare 解析，推荐使用 cloudflare 解析。

然后，打开博客的 GitHub Pages 仓库的设置页面，找到 GitHub Pages 的位置，在 Custom domain 位置填入自己的域名，保存。在浏览器输入自己的域名，看看能不能看到自己的博客。有时域名解析不能立即生效，等一段时间再看看，一般不会超过一天。

最后，保持永久的使用域名访问。（当然前提是，要及时为自己的域名付费。）上一步添加自己的域名时，自动在仓库里创建了一个 CNAME 的文件，这个文件名就是 CNAME，没有后缀，里面的内容就是你的域名。我们从本地部署博客到 GitHub 时，是没有这个文件的，因此下一次你再部署博客时，这个文件就没有了，不能持久。解决这个问题，只需要在 themes 文件夹下的主题目录下的 source 文件夹新建一个 CNAME 文件，文件内容写入自己的域名即可。默认的主题目录名是 landscape。

以后，写完一篇文章，只需要执行以下命令即可把文章部署到网站。

1hexo clean
2hexo g
3hexo d
