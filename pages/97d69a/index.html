  1. MAX_SAFE_INTEGER *知识点

    MDN 链接： MAX_SAFE_INTEGER (opens new window)

    JavaScript 中 Number 数据类型的最大安全整数为 $(2^{53} -1)$。

    之所以是这个数值，是因为 JavaScript 中的 Number 数据类型使用的是双精度浮点数，其范围为 $-(2^{53} - 1) <= x <= 2^{53} - 1$。

    因此，Number 数据类型的最小安全整数为 $-(2^{53} - 1)$。

    因为 MAX_SAFE_INTEGER 和 MIN_SAFE_INTEGER 是 Number 静态属性，因此可以直接在 Number 数据类型中使用它。

  2. value = +value *知识点

    此处的 + 是一元正号，一元正号运算符位于其操作数前面，计算其操作数的数值，如果操作数不是一个数值，会尝试将其转换成一个数值。如果它不能解析一个值，则计算结果为 NaN (opens new window).

    什么是一元正号：一元正号 (opens new window)

# 2. 数据类型（结构）练习二

原文：数据类型（结构）练习二 (opens new window)

原文代码：

const print = console.log;

function clamp(number, lower,upper) {
  number = +number;
  lower = +lower;
  upper = +upper; /* 1 */

  lower = lower === lower ? lower : 0; /* 2 */
  upper = upper === upper ? upper : 0;

  if (number === number) {
    number = number <= upper ? number : upper; /* 3 */
    number = number >= lower ? number : lower;
  }

  retrun number;
}

print(clamp(-1, 2, 3)); // => 2
print(clamp(2, -2, 3)); // => 2
print(clamp(4, -2, 3)); // => 3
print(clamp(1, 2, -3)); // => 2
print(clamp(2, 8, -1)); // => 8
print(clamp('a', 2, 3)); // => NaN
print(clamp(1, 'a', 3)); // => 1
print(clamp(1, 'a', 'b')); // => 0
print(clamp(1, '5', '6')); // => 5

# 5. 数据类型（结构）练习五

原文：查找数组元素的下标 (opens new window)

实现函数 indexOf，查找数组元素的下标

原文代码：

const print = console.log;

function indexOf(array,value,fromIndex) {
  const length = array === null ? 0 : array.length; /* 1 */
  if (!length) { return -1; }
  let index = fromIndex == null ? 0 : +fromIndex; /* 2 */
  if (index < 0) {
    index = Math.max(length + index, 0); /* 3 */
  }

  while (++index < length) {
    if (array[index] === value) {
      return index;
    }
  }
  return -1;
}

print(indexOf([1, 2, 3, 2], 2));// => 1
print(indexOf([1, 2, 3, 4, 2, 7, 8], 2, 2)); // => 4
print(indexOf([1, 2, 3, 2, 7], 2,-2)); // => -1

笔记：

indexOf 函数接受三个参数，array 是要查找元素的数组，value 是要查找的元素，fromIndex 是查找下标的起始位置，可以省略。JavaScript 函数传入参数的时候可以不按函数指定的参数个数传入参数，未传入的参数值为 null。

  1. 既然要查找数组元素的下标，因此首先必须要确保有数组传入，如果不传入数组，则查找不成立。这里用三元表达式判断并获取数组的长度，下一步如果数组长度为 0 ，则函数退出。
  2. 查找元素下标，不一定需要执行起始位置，因此第二步继续判断起始位置参数 fromIndex 是否存在，如果不存在，则从 0 开始查找。
  3. 有时候我们会从数组倒数某位查找元素，这时使用可以把起始位置定为负数，但在实际查找是起始还是从左边开始查找。这里的 if 语句将 index 值为负时对 index 值进行转换。

可以看到，数据处理函数首先要做好对传入参数数据合法性进行判断，使其符合预期要求，能够快速高效的进行数据合法性判断，也使得整个函数结构简洁明了。在这个函数中，判断数据合法性是都用了 === null 来判断，而 JavaScript 的数据类型中，只有 null === null ，而在后续的处理中，当遇到这种情况时，前面的变量被赋值为 0 ，因此也解决了这种情况。

# 6. 数据类型（结构）练习六

原文：按照给定大小切割成二维数组 (opens new window)

设计一个函数 chunk，按照给定的大小切割成二维数组

原文代码：

const print = console.log;

function chunk(array, size) {
  size = Math.max(size, 0); 
  const length = array == null ? 0 : array.length;
  
  if(!length || size < 1) {
    return [];
  }

  let index = 0;
  let resIndex = 0;
  const result = new Array(Math.ceil(length / size)); /* 1 */

  while (index < length) {
    result[resIndex++] = array.slice(index, (index += size)); /* 2 */
  }
  return result;
}

print(chunk(['a', 'b', 'c', 'd'], 2)); // => [['a', 'b'], ['c', 'd']]
print(chunk(['a', 'b', 'c', 'd'], 3)); // => [['a', 'b', 'c'], ['d']]

题目要求对传入的数组，按照给定的大小切割成二维数组，即切割后的二维数组每部分长度应是给定大小的长度，最后一部分长度不够也应分为一组。

有一个问题需要解决，就是怎么把这个数组切割成二维数组，我的想法是创建一个二维数组，二维数组的行为 2 ，列为源数组长度除以给定的大小，然后用 for 循环嵌套获得数组的每一个元素。然而看老师的代码，只用了一行就解决的这个问题，我认为关键就在于 Math.ceil 和 array.slice 这两个方法，因此要查资料，高明白这两个方法。

  1. 在 MDN 上查到 The Math.ceil() function returns the smallest integer greater than or equal to a given number. , 该方法返回大于等于给定值得最小正整数。因此这里 result 数组长度是二维数组单行的长度，查询犀牛书看到 JavaScript 没有真正意义上的多位数组，但可以通过数组的数组来得到多维数组。到这里还是不太明白，于是我就把代码跑了一遍，在 while 循环里加入 consolge.log 打印 result 数组，结果发现第一次循环时，将分割后的 array 第一部分传进了 result，第二次循环传入了第二部分。然后再打印 result 数组的长度，发现 result 数组的长度固定为 2 。这下我总算有点明白了，JavaScript 没有多位数组的概念，所谓数组的数组就是在数组里嵌套数组，而数组也是对象，计算多维数组时，数组里面的数组也是 object ，所以一个二维数组应该是 [object,object]，长度为 2 。如此一来，我感觉 const result = new Array(Math.ceil(length / size)); 这一句里定义数组的长度好像没有必要。
  2. 循环体里最主要的是 slice 这个数组属性的方法，MDN 的解释是 The slice() method returns a shallow copy of a portion of an array into a new array object selected from begin to end (end not included). The original array will not be modified. , 该方法返回一个以起始下标开始到终止下标结束（不包终止下标）的浅拷贝数组部分到新的数组，源数组不会被改变。这个拷贝过去的是一个数组，因此就完成了对数组的切割工作。

再次测试之后，发现 const result = new Array(Math.ceil(length / size)); 不能省略了定义数组长度。

# 7. 数据类型（结构）练习七 - 根据条件过滤数组

设计一个 filter 函数，根据条件过滤数组。第一个参数为数组，第二个参数为过滤条件函数。

原文代码：

const print = console.log;

function filter(array, condition) {
  let index = -1;
  let resIndex = 0;
  const length = array == null ? 0 : array.length;
  const result = [];

  while (++index < length) {
    const value = array[index];
    if (condition(value, index, array)) { /* 1 */
      result[resIndex++] = value;
    }
  }
  return result;
}

const users = [
  { 'user': 'barney', 'active': true },
  { 'user': 'fred', 'active': false }
]

print(filter(users,({ active }) => active)); // => [{ 'user': 'barney', 'active': true }]

const scores = [100, '90', '88', 97, 99];
print(filter(scores, (item) => item === +item)); // => [100, 97, 99]

# 8. 数据类型（结构）练习八 - 根据数组路径查找对象的属性值

原文：根据数组路径查找对象的属性值 (opens new window)

设计一个函数 getValue，第一个参数为对象，第二个参数为数组，第三个参数为可选的默认值。 把数组的元素集当成查找路径，找到对象中对应的属性值。

原文代码：

const print = console.log;

function innerGetValue(object, path) {
  let index = 0;
  const length = path.length;

  while (object != null && index < length) {
    object = object[`${path[index++]}`]; /* 1 */
  }
  return (index && index == length) ? object : undefined; /* 2 */
}

function getValue(object, path, defaultValue) {
  const result = object == null ? undefined : innerGetValue(object, path); /* 3 */
  return result === undefined ? defaultValue : result;
}

print(getValue({ 'a': [{ 'b': { 'c': 3}}]}, ['a', '0', 'b', 'c'])); // => 3
print(getValue({ 'a': [{ 'b': { 'c': 3}}]}, ['a', 0, 'b', 'c'])); // => 3
print(getValue({ 'a': [{ 'b': { 'c': 3}}]}, ['a', 1, 'b', 'c'])); // => undefined
print(getValue({ 'a': [{ 'b': { 'c': 3}}]}, ['a', 1, 'b', 'c'], 0)); // => 0
print(getValue({ a: { b: { c: 1 }}}, ['a', 'd', 'c'])); // => undefined
print(getValue({ a: { b: { c: 1 }}}, ['a', 'd', 'c'], 0)); // => 0

笔记：

此函数的作用是通过给定的数组路径去对象中查找对象的属性值。我认为这里面考察了对象和数组的基础知识，JavaScript 里对象里可以嵌套数组，数组里也可以嵌套对象，即数组可以作为对象属性值，对象也可以作为数组元素。因此，首先我们要知道所查找的对象的结构，其次我们的数组路径要符合对象结构层次。

先来看代码，因为这个题目如果给我放在这里，让我写，我是无从下手的。

  1. 首先，定义了一个 innerGetValue(object, path) 函数，接收 object 、path 两个参数，object 是要查找的对象，path 是查找路径数组。该函数 while 循环体内，判断条件有两个，一个是 object 不能为空，如果是一个空对象，就没有查找的必要了，另一个是数组路径长度要大于 0 ，如果数组长度为 0 ，也就没有查找必要了。只有这两个条件都满足后，开始执行循环体。循环体对对象重新赋值，这里使用的 ${path[index++]} 是模板字符串 (opens new window)，这里是将 path 数组元素转换成字符串，传递给对象属性。如果 object 并未此属性，那么 object 就会是 undefined，循环判断条件 undefined != null 为 false，循环体结束执行。
  2. 函数 innerGetValue 返回值先对 index 进行了判断，当所有条件都为真是，循环体执行完毕，index 为 true，index 等于 length，返回对应路径对象属性值。反之，循环体中断执行，index 不等于 length，即没有找到对应路径对象属性值，返回 undefined。
  3. 主函数 getVaue(object, path, defaultValue) 接收三个参数，最后一个参数为可选的默认值。主函数体内，首先对对象进行了非空判断，只有先传入一个非空对象才能进行后续操作。对对象的具体处理工作通过调用 innerGetValue 函数执行，执行完毕后返回结果。

在主函数 getValue 和内部处理 innerGetValue 函数中，分别对参数进行了类型的判断，且是各司其职。主函数 getValue 的目的就是要获得结果，要保证输入的对象不能为空，然后把剩下的事交给内部处理函数 innerGetValue 就可以了，它处理完了会给主函数一个值，主函数只需要拿到这个值再做一次判断，并把最终结果返回就可以了。

内部处理函数 innerGetValue 主要工作是查找给定路径的对象属性值，因此必须保证路径数组有值，这里做了一个非常巧妙的处理，首先获取路径数组的 length 值，然后定义了一个值为 0 的 index，再判断 index < length，如果 length 为 0 ，那么循环体根本不会执行，返回值处理时因为 index 为 0,而返回 undefined，最终返回值也为 undefined。

innerGetValue 函数短短几行代码，却包含了很大的信息量，确实是精妙。

# 9. 数据类型（结构）练习九 - 基础数据类型的隐式转换

原文：基础数据类型的隐式转换 (opens new window)

笔记：

编程语言中，两种不同数据类型的值没有可比性，但有些编程语言可以对不同数据类型的值进行比较。

能进行比较的雀巢是将一个值转换成为另一个值得数据类型，或者将两个不同数据类型的值转换成同一个数据类型， 然后再进行比较。

# 三种隐式类型转换

JavaScript 中，基本数据类型（number, boolean, string, null, undefined）可能发生 ToNumber, ToBoolean, ToString 三种隐式类型转换情况。

# ToNumber(转换为数字)

所有基本类型都可以转换成为 number 类型，可以通过单目运算符(+)操作来验证转换结果。

+1; // => 1
1

# ToBoolean(转换为布尔)

所有基本类型都可以转成为 boolean 类型，可以通过 Boolean() 方式验证转换结果。

Boolean(1); // => true
1

# ToString(转换为字符串)

所有基本类型都可以转换成为 string 类型，可以通过 String() 方式验证转换结果。

String(2); // => '2'
1

# 发生转换的场合

隐式转换一般发生在逻辑运算中，不同逻辑运算符之间的类型转换会有差异。

  • 两个 string 类型值相加，得到拼接的字符串
  • number 和 string 相加，number 会转换成 string
  • 两个 string 类型值相减，会将 string 转换为 number，做减法操作，如果一个不是数字字符串，会得到 NaN
  • ToBoolean 一般会用在需要真假值得场合，如 if, switch
  • 比较值如果有一个是 number 类型，另一边也会进行 ToNumber 转换
  • 任何值和 NaN 比较都为 false
  • 如果逻辑比较的两边都是 string 类型，会取首字符的 ASCII 码值进行比较
    • 在编程时，避免两个字符串的硬比较
    • 避免与 NaN 的比较，特别是隐式转换时得到的 NaN

如果编程语言具备隐式的类型转换，就可以说该语言是弱类型语言 相应地，强类型语言则不太容易容许隐式的类型转换。JavaScript 是弱类型语言。

# 小课堂：对象的隐式转换

原文：对象的隐式转换 (opens new window)

对象的隐式类型转换，执行的是 ToPrimitive 操作。

ToPrimitive 的目的是将一种数据类型转换为基础数据类型。 基础数据类型进行 ToPrimitive 转换还是得到本身 object 类型进行 ToPrimitive 操作会转成基础数据类型。

ToPrimitive 操作得到具体值是通过 object 的两个内置函数返回的：object.toString() 和 object.valueOf() 。

ToPrimitive 中有三种转换暗示：string, number, default。

要搞清楚对象在进行类型转换求值时，是依赖 toString() 还是 valueOf()，要记住一些原则：

对 object 进行类型转换时，会进行 ToPrimitive 操作。 ToPrimitive 会依赖对象的 toString() 或 valueOf() 方法返回的结果。 ToPrimitive 在转换过程中会根据 hint 决定调用 toString() 还是 valueOf()

  • hint 是 string

首先，调用对象的 toString()方法，如果值是基础数据类型则返回。 否则，调用对象的 valueOf() 方法，如果值是基础数据类型则返回。 否则，抛出TypeError错误

  • hint 是 number

首先，调用对象的 valueOf()方法，如果值是基础数据类型则返回。 否则，调用对象的 toString() 方法，如果值是基础数据类型则返回。 否则，抛出TypeError错误

  • hint 是 default

与 hint 是 number 的逻辑相同

对象相加，hint 为 default ，对象相减，hint 为 number ；对象进行逻辑运算，hint 不是 default 就是 number。

const print = console.log;

function slice(array, start, end) {
  let length = array == null ? 0 : array.length; /* 1 */
  if (!length) {
    return [];
  }
  /* 2 */
  start = start == null ? 0 : start;
  end = end === undefined ? length : end;
  /* 3 */
  if (start < 0) {
    start = -start > length ? 0 : (length + start);
  }
  end = end > length ? length : end;
  if (end < 0) {
    end += length;
  }
  length = start > end ? 0 :(end - start);

  let index = -1;
  const result = new Array(length);
  while (++index < length) { /* 4 */
    result[index] = array[index + start];
  }
  return result;
}
