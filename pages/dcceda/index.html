      MSSQL数据库重置表ID

      本文解决 MS SQL SERVER 数据库删除表数据后，如何重置 ID 的问题。

      # 方法一、用 TRUNCATE

      语句

      TRUNCATE TABLE table_name
      1

      可以删除表内所有值并重置标识值

      # 方法二、用 DBCC CHECKIDENT

      语句

      DBCC CHECKIDENT('table_name',RESEED,new_reseed_value)
      1

      注意：如果表内有数据，则重设的值如果小于最大值可能会出现问题。

      # 关于标识列其他命令

      # 1. 判断一个表是否具有标识列

      语句

      SELECT OBJECTPROPERTY(OBJECT_ID('table_name'),'TableHASIdentity')
      1

      如果有，返回1,否则返回0

      # 2. 判断某列是否是标识列

      语句

      SELECT COLUMNPROPERTY(OBJECT_ID('table_name'),'column_name','IsIdentity')
      1

      如果是标识列，返回1,否则返回0

      # 3. 查询表标识列列名

      语句

      SELECT COLUMN_NAME FROM INFORMATION_SCHEMA.columns
  WHERE TABLE_NAME='table_name'
  AND COLUMNPROPERTY(OBJECT_ID('table_name'),COLUMN_NAME,'IsIdentity')=1
      1
      2
      3

      # 4. 获取标识列的种子值

      语句

      SELECT IDENT_SEED('table_name')
      1

      # 5. 获取标识列的递增值

      语句

      SELECT IDENT_INCR('table_name')
      1

      # 6. 获取表中最后生成的标识值

      语句

      SELECT IDENT_CURRENT('table_name')
      1

      如果数据表是刚刚创建，并未插入数据，此时使用 IDENT_CURRENT 函数得到的值未标识列的种子值。
