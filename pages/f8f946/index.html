    Windows + Linux 双系统如何彻底卸载 Linux？

    在 Windows 系统的电脑中安装 Linux 是一件很方便的事情，能让我们有流畅的 Linux 体验。然而出于种种原因，使用 Linux 作为桌面环境，总会存在很多问题，用了一段时间之后因为种种原因，不得不继续回到 Windows 的怀抱。这个时候，安装 Linux 时占据的磁盘分区我们就想拿回来。很多人会想当然的，直接删除掉 Linux 磁盘分区，然而删除之后，重启电脑却进步了 Windows，这个时候可能会有点慌。其实不用怕，我们要相信，我们所碰到的问题都不是个例，互联网上已经有了解决方法。这篇文章就是综合在写在 Linux 时存在的不同情况，将互联网上的解决方法集合而成。

    # 一、正常卸载方式

    # 1. 判断启动方式

    在 Windows 中以管理员身份打开 powershell，输入 bcdedit，找到 Microsoft Windows 的 path 行，如果是 winload.efi 就是 uefi 引导，若为 exe 就是 legacy 引导。

    # 2. legacy 卸载方法

    使用 mbrfix 工具。

    下载 Mbrfix 软件，解压到 C:\windows\system32 文件夹中。

    以管理员身份打开 powershell，如果是 32 位系统在命令提示符中输入 Mbrfix /drive 0 fixmbr /yes，如果是 64 位系统则要输入 Mbrfix /drive 0 fixmbr /yes。

    重启，能正常进入 Windows 则表示卸载成功。

    删除 Linux 分区。

    在 Windows 10 开始图标上点击鼠标右键，进入磁盘管理(disk management)，删除 Linux 分区。再将删除的空白分区扩展给 Windows 使用。

    # 3. uifi 卸载方法

    下载 easyUEFI 工具，找到 Linux 系统发行名，例如 ubuntu，将其删除。重启电脑，删除对应 Linux 分区即可。

    # 二、暴力卸载 Linux

    在 Windows 下直接删除 Linux 磁盘分区。

    重启电脑，会进入一个 GRUB 引导界面，在光标位置输入 exit 退出 GRUB 引导界面，连续点击 F12，进入 Windows 引导界面，选择 Winodws Boot Manager 进入 Windows 10。

    进入 Windows 10 后，以管理员身份打开 powershell，输入 mountvol M: /s，按 Enter 键，将 EFI 分区挂载到 M 盘。

    输入 M: 按 Enter 键进入 M 盘。输入 ls 查看盘符内文件，会看到一个 EFI 文件夹。

    输入 cd EFI 进入 EFI 文件夹，继续输入 ls 查看该文件夹下文件，会发现有 Linux 的文件夹，如 ubuntu。

    执行命令 rmdir ubuntu 按 Enter 键，提示该文件夹下有子文件夹，问是否删除，输入 A 删除所有。

    现在 重启电脑试一下，就可以正常进入 Windows 了。
